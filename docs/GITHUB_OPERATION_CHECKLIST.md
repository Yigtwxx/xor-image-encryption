# GitHub Operations Checklist (Manual)

Some visibility tasks require repository admin actions in GitHub UI.

## 1) About and Topics

Set repository description to:

`Seed-based XOR image encryption for deterministic dataset masking and computer vision preprocessing.`

Set topics:

- `image-encryption`
- `xor`
- `computer-vision`
- `dataset-anonymization`
- `python`
- `numpy`
- `pillow`
- `image-processing`
- `reproducibility`
- `cryptography-education`

## 2) Discussions

Enable Discussions and pin these three threads:

- Showcase your encrypted output
- Q&A for integration in CV pipelines
- Roadmap voting

## 3) Pinned roadmap issue + milestone

- Create milestone: `v0.2.x`
- Add goals: CLI polishing, batch mode, benchmark reproducibility
- Open an issue from `ROADMAP.md` and pin it

## 4) Releases

- Create tag + release `v0.1.0` using `docs/RELEASE_NOTES_v0.1.0.md`
- After onboarding/doc tweaks, create `v0.1.1`

## 5) Weekly metrics

Track these values from GitHub Insights every week:

- Unique visitors
- Clones
- Stars
- Returning visitors
